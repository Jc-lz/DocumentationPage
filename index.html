Html

HTML (Hyper text Markup Language) es la construcción más básica de un sitio web. Definiendo el sentido y la estructura del contenido Web. Otras tecnologias ademas de HTML son generalmente usadas para describir la apariencia de una página web o su funcionalidad.

"HyperText" se refiere a los enlaces que conectan una página web con otra, dentro de la misma página o entre varias páginas web. Los enlaces son un aspecto fundamental de la Web. Subiendo contenido al internet y enlazandolo a distintas páginas creadas por otras personas te convertiras en un participante activo en el mundo de la red mundial.

HTML usa marcas para mostrar texto, imagenes y otro tipo de contenido en el buscador Web. Estas marcas incluyen elementos especiales que veremos más adelante.

a

El elemento "a" crea hipervinculos a otras páginas web, archivos, secciones de la misma página web, direcciones de correo o cualquier otro URL.

Atributos

href

Contiene un URL que apunta a un hipervinculo.

El nombre del fragmento del URL viene de la marca hash (#), la cual especifica una ubicación interna (un ID o un elemtento HTML).

hrefland

Este atributo indica en lenguaje humano la funte enlazada, es puramente divisora, sin una funcionalidad constructora, aloja valores que son determinados por BCP47

referrerpolicy

Indica que referencia envia al buscardor:

  • 'no-referrer' significa que Referer: no será enviado

  • 'no-referrer-when-downgrade' significa que Referef: será enviado el header cuando se navega sin un link de orgen,

    • 'origin' significa que el 'referrer' será el origen de la página, no incluye información antes del dominio.

    • 'strict-origin-when-cross-origin'

    • 'unsafe-url' significa que el referrer será incluido en el origen y el camino, pero no el fragmento, las contraseñas, o los usuarios. Esto es inseguro porque pueden extraer la información de URLs seguros a URLs inseguros.

    Artcile

    El elemento article representa una composición auto-contenida en un documento, páigna, aplicación o sitio que, puede ser reutilisable.

    Notas de uso

    • Cada 'article'debe ser identificado, tipicamente se incluyen etiquedas como h1...h6 como hijo del elemento 'article'

    • Cuando un un elemento tipo 'article' esta anidado, el elemento interno representa un elemento relacionado con otro elemento, el comentario de un post de un blog puede ser un 'article' anidado en otro 'article' representando el post del blog.

    • La publicación de un elemento tipo 'article' puede ser descrita usando el atributo 'datetime' atributo del elemento 'time'.

    Button

    El botón representa un botón clickleable que puede ser usado en 'forms' o en cualquier documento que necesite de una simple funcionalidad de un botón. Por defecto, los botones de HTML son presentados en forma similar a la plataforma en la que se esta corriendo, esta apariencia puede ser cambiada usando CSS.

    Atributos

    Los atributos de este elemento incluyen atributos globales.

    • Autofocus HTML5

      Este atributo booleano permite especificar que el botón debe tener un input focus cuando la página es cargada amenos que el usuario no lo quiera así.

    • Disabled

      El uso de este atributo indica que el usuario no puede interactuar con el botón. Si el atributo no lo especifica el bontón heredará las caracteristicas del elemento contenedor.

    Div

    El contenedor divisorio div es un contenedor generico, no tiene un efecto en el diseño a menos que sea estilizado usando CSS.

    Como un "contenedor puro" el elemento div no representa nada, es utilizado para agrupar contenido que puede ser facilmente estilizado usando los atributos class o id marcando una sección de un documento que será escrita en un lenguaje diferente.

    Notas de uso

    • El elemento div debe ser usado unicamente cuando otro elemento (como article o nav) estan presentes, ya que el elemento div no tiene un sentido semantico.
    Form

    El elemento form representa una sección del documento que contiene controles interactivos para enviar información a un servidor web.

    Es posible las clases :valid o :invalid para estilizar el elemento form

    Atributos

    • Action

      El URI del programa puede procesar la información del formulario, este valor puede ser sobreescrito por el atributo formaction en el elemento button o input

    • Name

      El nombre en el formulario. En HTML 4, es despreciado. Debe ser usado entre los documentos del formulario y no en un valor string vacio.

    Header

    El header presenta el contenido introductorio, tipicamente un grupo introductorio o ayudas de navegación. Puede contener algunos elementos como un logo, una barra de busqueda, el nombre del autor y otros más.
